DRAGON COLLECTION TROUBLESHOOTING CHECKLIST
============================================

If dragons are not disappearing on tap, check the following:

1. SCENE SETUP
   ‚òê ScoreManager exists in scene (with TextMeshPro UI for score display)
   ‚òê Canvas exists with EventSystem component
   ‚òê Main Camera is tagged as "MainCamera"
   ‚òê Dragons are spawned in the scene (check Logs for spawn messages)

2. DRAGON PREFAB SETUP
   ‚òê Dragon prefab has "Collectible" script component
   ‚òê Dragon prefab has SphereCollider component (not disabled)
   ‚òê Collider is set to: IsTrigger = false
   ‚òê Collider Radius = 5
   ‚òê monsterName field is set (e.g., "Dragon Nightmare")

3. PHYSICS LAYER SETUP
   ‚òê Dragons are on Default layer (or correct physics layer)
   ‚òê Camera has correct culling mask
   ‚òê No layer collision matrix issues preventing raycast

4. SCRIPT DEBUGGING
   ‚òê Open Console (Window > TextMeshPro)
   ‚òê Tap/Click a dragon
   ‚òê Look for log messages:
      - "[Collectible] Initialized: ..." - Dragon script is ready
      - "[Collectible] ‚úì TOUCHED: ..." - Touch was detected
      - "[Collectible] üéâ COLLECTED: ..." - Collection successful

5. INPUT DEBUGGING
   ‚òê Add InputDebugger script to any GameObject in scene
   ‚òê Tap/Click anywhere on screen
   ‚òê Check Console for raycast hit information
   ‚òê Verify dragon colliders are being hit

6. UI BLOCKING CHECK
   ‚òê Make sure Canvas is not blocking input
   ‚òê Canvas RaycastTarget should be FALSE for non-interactive elements
   ‚òê Test in a scene without UI overlay

7. COMMON ISSUES & FIXES

   Problem: "No main camera found!" error
   Fix: Ensure main camera is tagged as "MainCamera"

   Problem: Raycast hits nothing
   Fix: 
   - Check collider is not disabled
   - Check collider radius (5 is usually good)
   - Verify dragon is visible in game view
   - Check layer collision matrix

   Problem: Raycast hits but doesn't collect
   Fix:
   - Verify ScoreManager instance exists
   - Check Collectible script is not missing
   - Check monsterName is not empty

   Problem: Dragon disappears but score doesn't update
   Fix:
   - Verify ScoreManager.instance is not null
   - Check TextMeshProUGUI component is assigned
   - Verify scoreText reference in inspector

2. Open Console window
3. Tap/Click on visible dragon
4. Check Console for debug messages
5. Verify dragon disappears and score increments
